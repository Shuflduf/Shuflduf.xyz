Functionality

Note: This is the PLAN of what I want rcket to be like. Only a small part of this is actually implemented.

First, you define a list of tokens for the lexer using Rust types, then you use a define macro on it to turn it into a lexer. That's it. You can just use it to turn code into a list of tokens.

The AST parser is just as easy to use. You just define how some tokens fit together, using regex or direct identifiers, and rcket figures it out!
